NEWAVES APP

UX/UI Design

December 2023

Project Vision

Newaves is a streaming audio app focused on local content from 4 different countries, Portugal, North Macedonia, Slovakia and Croatia.

Challenges

a) something that users are familiar with, but have some own identity too.

b) Create an efficient, effective and satisfactory product, following the usability rules.

c) Make the main tasks easy to complete for everyone.

Define the goals

This was an academic project, so, for this one, we already had our goals well established and defined. Each group of users should have their own permissions.

Group 1 (Normal User):

Listen, upload, edit, rate and comment on content, create playlists and follow profiles.

Group 2 (Administrator User):

Validate content, manage channels, create channel profiles and have access to stats.

Branding

For this project I also created a visual identity, where I used 3 elements to guide me while building the logo, the red color, because it’s the one in common in all of the flags of the 4 countries of this project, the waves shape, to represent the audio, and the initials “N” and “W” because of the app’s name.

User Testing

After all the high-fidelity prototype was built, it was time to test the app with some possible real users. A selection was made based on some criteria, taking into account some requirements that real users would have. The tests were made in a presential environment, where users, one by one, were asked to complete some tasks while, at the same time, thinking out loud during the entire process. All those tests allowed to discover some points that needed improvements.

Language

A very important option for this app to work well among all potential users, as its main markets are 4 different countries, each with its own language, it is very important to have this support for each of them. As there are 4 different countries, options with the languages ​​of each of them have been added.

Confirmation
Messages

This suggestion fits into one of the 5 attributes of usability according to Nielsen, the “Low error rate”, which concerns doing everything possible so that the user makes as few mistakes as possible, and one way to prevent this is through these messages, which makes the user have to make an extra click before completing an important task, ensuring that the user really wants to perform it.

Help Center

Not everyone has the same knowledge and experience with technology, so we cannot take for granted that everyone understands how to use the product, so it is always good to have a help center that allows users to learn how to use the app.
